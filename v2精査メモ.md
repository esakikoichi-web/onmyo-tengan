# 陰陽天眼 v2.0 精査メモ（商品化観点の監査）

対象：アップロードされた v2.0（2.7MB / 23,155行 / 関数223個）
タイトル「四柱推命 自己理解・経営者リフレクションツール v2.0」

## 結論（先に）

**これは占いアプリではなく、四柱推命エンジン＋Claudeで動くB2B人材・組織診断アプリ。**
中身の作り込みは想像以上に本格的で、「商品の核」はほぼ完成している。
足りないのは**「本物のサーバ・認証・課金・APIキー秘匿」＝商用インフラだけ**。
UIの課金ロックや会員ログインは"見た目"は出来ているが、実体は全てブラウザ内で
bypass可能な張りぼて。ここを本物に差し替えれば売り物になる。

---

## 1. 実際に作り込まれている資産（強い）

- **四柱推命エンジンが本物**：立春・節入りの補正あり（`節入`/`立春` 多数）。年月日時から
  正しく命式を出せている。営業版と同系の自前計算。
- **画面は約10本の本格構成**：
  `login / menu / chatbot(AI陰陽師) / dash / job・jobForm・jobMatch(採用) /
   form(入力) / compare / keiei(経営者) / compat(相性) / report`
- **レポートが巨大**：`rp-page` が **133ページ分**のテンプレート。気質・大運・有名人
  マッチング・健康養生・経営者の心得・投資相性など多面的。
- **Claude連携が4系統**：
  | 関数 | 用途 | モデル | max_tokens |
  |---|---|---|---|
  | generateAIAnalysis | 命式総合分析（主力） | Haiku 4.5 | 2000 |
  | generateInterviewQuestions | 面接質問生成 | Haiku 4.5 | 1500 |
  | autoAnalyzeDept | 部署の自動分析 | Haiku 4.5 | 400 |
  | generateKeieiAI | 経営者運気 | Sonnet 4 | 600 |
  - プロンプトは「自己理解・キャリアリフレクションの専門コーチ」等、**占いを前面に
    出さない設計**。ポジショニングが一貫していて良い。
- **B2Bのデータモデルが既にある**（localStorageキー）：
  `会社(meiseki_saved_company) → 部署(meiseki_depts_/deptdesc_/deptreqs_)
   → 社員(meiseki_staff_) → 求人(inyou_jobs_)`。組織を丸ごと扱う設計。

## 2. コストの現実（朗報）

主力がHaiku 4.5中心のため、**1レポートあたりのAPI実費は数円〜十数円程度**。
以前「無制限だと赤字」と警告したが、現状のモデル選定なら**コストは実質的な制約に
ならない**。無制限使い放題でも十分成立する。※将来Opus等に上げる時だけ要再計算。

## 3. 張りぼて＝本物に差し替えが必要な部分（ここが全て client-side で bypass可能）

- **ログインがデモパスワード `1234` のハードコード**（`デモ版パスワード：1234`）。
- **会員認証がURLパラメータ頼み**：`?from_member=...&company=...` を信じて
  `onmyo_authed_company` に保存するだけ。URLを叩けば誰でも"会員"になれる。
- **課金ロックがCSSモザイク（blur）だけ**：`free-trial` クラスで視覚的にぼかすのみ。
  DevToolsでクラスを外せば全部見える。決済連携は無し（Stripe等の文字列は"有名人データ"で、実装ではない）。
- **Claude APIキーがユーザー手入力**（`inyou_apikey`/`yyg_apikey` を localStorage）。
  お客さんに配れない。商品化では成立しない。
- 全データがブラウザ内 localStorage のみ＝端末を変えると消える／共有できない。

→ つまり**「価値ある中身」と「それを守る壁」が分離できていない**。
  中身は良いのに、今は誰でもタダで全部使える状態。

## 4. 未完成・要整理

- `準備中` / `ダミー` マーカーが数か所（軽微。全体は概ね完成）。
- 2.7MBの単一HTMLに実験機能も同居。**正式機能として売るモジュールの取捨選択**が必要。

## 5. 商品化ロードマップ（v2.0前提・B2B）

土台が出来ているので、やるべきは「張りぼて→本物」の置換に集中する。

1. **APIキーをサーバに秘匿**：Claude呼び出しをSupabase Edge Function経由に。
   キーはあなたの1本をサーバに置き、課金企業だけがプロキシ経由で叩く。
   （＝コピー対策とコスト管理を同時に解決）
2. **本物の認証**：デモパスワード/URLパラメータを廃し、企業アカウント＋サーバ発行トークンに。
3. **本物の課金ゲート**：CSSモザイクの裏に、サーバ側の契約状態チェックを入れる。
   B2Bなら**請求書払い（銀行振込）**で占い決済審査を回避できる公算大。
4. **データをサーバ保存**：会社・部署・社員・求人をPostgresへ。端末越しに共有。
5. **売るモジュールの確定**：採用マッチング／自己理解／チーム相性／経営者運気から
   正式機能を選び、実験部分を隠す。
6. **法務**：従業員の生年月日＝個人情報。B2Bは"従業員本人の同意"も必要。
   特商法（または請求書払いなら契約書）・規約・プライバシーポリシー・「参考/娯楽」の免責。

## 6. 価格・売り方（再掲・v2.0版）

- 中小企業（10〜50名）の経営者・人事向け、**企業単位の従業員数課金**。
  例：〜10名 ¥19,800／〜30名 ¥39,800／〜50名 ¥69,800。
- 主役の入口は**採用マッチング**（採用失敗の損失は数百万＝ROIで高単価が通る）。
- 見せ方は現状維持で「性格の設計図／組織アセスメント／AI陰陽師」。占いは奥に隠す。
