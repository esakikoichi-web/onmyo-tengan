# Googleスプレッド自動鑑定（=鑑定 関数）

Instagram / Threads で集めたお客様の生年月日を貼るだけで、**鑑定文（G列）を自動生成**するための Google Apps Script です。
これまで1人ずつ手で書いていた「返信した内容」を、関数1つで自動化します。

Web版「陰陽天眼」の一括鑑定と**同じ結果**が出ます（生年月日の「日」から人物の芯を割り出す方式。性別は不要）。

---

## できること

| 入力（生年月日） | 出力（自動で入る鑑定文） |
| --- | --- |
| `2002/08/06` | 太陽タイプ　真ん中で輝きたい人 … |
| `1988/6/4` | 刃タイプ　最短で結果を斬り取る人 … |

- 日付は `2002/08/06` `1988/6/4` `1990年3月5日` など、書き方がバラバラでもOK
- 「女 2002/08/06 13:51」のように**性別や時間が混ざっていても**、生年月日だけ拾って鑑定します
- セルが日付型でも、文字列でも動きます

---

## 導入手順（5分・1回だけ）

1. 鑑定を入れたい **Googleスプレッドシートを開く**
2. 上のメニュー **［拡張機能］→［Apps Script］** をクリック
3. 開いたエディタの中身を**すべて消して**、この `鑑定.gs` の中身を**丸ごと貼り付け**
4. **保存**（フロッピーのアイコン、または `Ctrl+S` / `⌘+S`）
5. スプレッドシートに戻り、鑑定文を入れたいセルに次を入力：

   ```
   =鑑定(E2)
   ```

   （`E2` は生年月日が入っているセル。あなたのシートに合わせて変えてください）

6. 初回だけ「承認が必要です」と出たら、案内に沿って**自分のGoogleアカウントで許可**してください
   （外部に送信するものは何もありません。計算はすべてシート内で完結します）

---

## 使い方いろいろ

### 1人ずつ
```
=鑑定(E2)
```

### まとめて全員分（おすすめ）
G2 に次を1つ入れるだけで、E2〜E100 の全員分が縦に自動で並びます：
```
=鑑定(E2:E100)
```
> ※ この使い方の場合、G3以下は空けておいてください（自動で埋まります）。
> 新しいお客様をE列に足すたびに、鑑定文が自動で増えます。

### 直接入力
```
=鑑定("1988/6/4")
```

### 英語名でも同じ
```
=ONMYO(E2)
```

## よくある質問

**Q. 元の手入力の鑑定文と少し表現が変わる？**
A. はい。この関数は Web版「陰陽天眼」と同じ、日主ベースの短い特徴文を出します。そのまま送ってもよいですし、気になる所だけ手で直してもOKです。土台が一瞬で埋まるので、ゼロから書くより圧倒的に速くなります。

**Q. 性別や生まれた時間は入れなくていい？**
A. この「特徴」鑑定は生年月日だけで出ます。より深い（大運・時柱まで観る）鑑定が必要な人は、Web版「陰陽天眼」や、青龍孔一への直接相談につないでください。

**Q. Instagram/Threadsの返信を自動でシートに取り込めない？**
A. そこはMeta側の制限で自動化が難しい部分です。返信の**転記は手作業のまま**でも、一番時間のかかる「鑑定文を書く」工程がこの関数で消えます。

## 仕組み（技術メモ）

- `_dayStem(y,m,d)`：基準日からの経過日数で十干（日主）を算出。Web版 `dayP()` と同一。
- `KCHAR` / `KMOTO` / `KOGLEAD`：Web版 `index.html` と同一のデータテーブル。
- `_tokucho()`：Web版の一括鑑定 `bTokucho()` と同一の文面ロジック。

データテーブルや文面を更新したいときは、`index.html` 側と `鑑定.gs` 側の両方を合わせて直してください。
